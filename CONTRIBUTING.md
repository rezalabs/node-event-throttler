# Contributing to Node Event Throttler

First off, thank you for considering contributing to Node Event Throttler! It's people like you that make the open source community such a great place to learn, inspire, and create.

## Code of Conduct

By participating in this project, you agree to abide by our Code of Conduct.

## How Can I Contribute?

### Reporting Bugs

*   **Check the FAQ** to see if your issue is already addressed.
*   **Search existing issues** to see if the bug has already been reported.
*   **Use the Bug Report template** when opening a new issue.
*   **Include as much detail as possible**, including steps to reproduce, environment information, and logs.

### Suggesting Enhancements

*   **Check existing issues** to see if the feature has already been requested.
*   **Use the Feature Request template** when opening a new issue.
*   **Describe the use case** and why this enhancement would be beneficial.

### Pull Requests

1.  Fork the repository.
2.  Create a new branch for your feature or fix.
3.  Write tests for your changes.
4.  Ensure all tests pass (`npm test`).
5.  Run the linter (`npm run lint`).
6.  Submit a pull request with a clear description of the changes.

## Development Setup

1.  Clone your fork.
2.  Install dependencies: `npm install`.
3.  Ensure you have a local Redis server running for integration tests.
4.  Run tests: `npm test`.

## Style Guide

We use the [StandardJS](https://standardjs.com/) style guide (with 4-space indentation). Please run `npm run lint` before submitting your PR.
